# 당신의 고양이 디자인 원칙

이 문서는 이후 Codex 작업에서 계속 참고할 핵심 기준이다.

## 핵심 감성

- 잘 만든 게임보다 이상한 고양이 반응이 먼저다.
- 예쁜 캐릭터보다 묘하게 기억나는 표정이 중요하다.
- 유저가 느껴야 하는 감정은 "뭐야 이거 ㅋㅋ", "아 귀엽네", "또 들어와볼까?"다.

## 화면 원칙

- 한 화면에는 고양이 1마리만 크게 보여준다.
- 버튼은 1~2개만 둔다.
- 문장은 짧은 반응형 문구로 쓴다.
- 설명, 세계관, 안내 문구를 늘리지 않는다.

## 고양이 원칙

- 그림판으로 그린 듯한 손그림 느낌을 유지한다.
- 삐뚤삐뚤한 선, 어색한 표정, 허술한 비율을 허용한다.
- 고퀄 일러스트, 전문 작가 느낌, 과한 명암은 피한다.

## UI 원칙

- 크림톤 배경과 연핑크 버튼을 기본으로 한다.
- 테두리는 얇은 검정 또는 갈색 선을 사용한다.
- 카드와 버튼은 둥글지만 너무 세련되게 만들지 않는다.
- 화려한 파티클, RPG식 연출, 재화 시스템 느낌을 금지한다.

## 정보 표시 원칙

- 희귀도는 내부 데이터로만 유지하고 UI에서 크게 강조하지 않는다.
- 컬렉션은 도감 게임처럼 빽빽하게 만들지 않는다.
- 홈 화면 기본 요소는 앱명, 타이머, 고양이, 이름, 짧은 대사, 쓰다듬기 버튼, 내 고양이들 버튼까지다.
